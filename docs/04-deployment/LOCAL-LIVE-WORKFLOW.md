# Local Development with Live Database

This workflow uses **live cPanel database** from your local machine. No `db:pull` needed.

---

## Setup (one-time)

### 1. Local `.env` — use live database

```env
# Use remote (live) database
DB_CONNECTION=mysql_remote

REMOTE_DB_HOST=system.khbevents.com
REMOTE_DB_PORT=3306
REMOTE_DB_DATABASE=khbevents_aebooths
REMOTE_DB_USERNAME=khbevents_admaebooths
REMOTE_DB_PASSWORD=your_password

APP_URL=http://127.0.0.1:8000
```

### 2. Enable Remote MySQL in cPanel

- cPanel → **Remote MySQL** → Add your IP (`202.62.59.252` or `%`)

---

## Daily workflow

### Local development

```bash
php artisan serve
```

- Code runs locally
- Database is live (cPanel)
- Changes to data affect the live site

### Push code to GitHub (normal)

```bash
git add .
git commit -m "Your message"
git push origin main
```

- `.env` is ignored — never pushed
- Code, assets in repo, migrations — all pushed as usual

### Deploy to live server

1. On the server (cPanel terminal or SSH):

```bash
cd ~/system.khbevents.com   # or your project path
git pull origin main
composer install --no-dev --optimize-autoloader
php artisan migrate --force
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

2. If you use Vite/npm assets:

```bash
npm ci
npm run build
```

3. Ensure `storage` symlink exists:

```bash
php artisan storage:link
```

---

## Summary

| Action | What happens |
|--------|--------------|
| **Local dev** | Local code + live DB |
| **Git push** | Code only (no .env, no secrets) |
| **Deploy** | `git pull` on server + migrations + cache |

## Assets & images

- **Code assets** (CSS, JS from `npm run build`) — in repo or built on deploy
- **User uploads** (avatars, floor plan images) — stay on server, generated by live site
- **Static images** in `public/` — in repo and deploy with code
